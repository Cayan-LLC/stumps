What is Stumps?

Stumps is an HTTP proxy and testing tool that lives on the backend of your DEV and QA environments. It lets you record the traffic from any web server, and use the recordings to create mocks (a "stump") for all of your integration tests without having to write a single line of code.

When a request you created a stump for comes through, it will return a known response. Otherwise, Stumps will act like a proxy and forward the traffic to the original website.

Why use Stumps?

We created stumps as a way to test our own applications. In our environment, automated integration tests constantly communicate with dozens of third party services. Network hiccups, flakey test accounts, and solar winds were constantly causing our tests to fail.

Stumps provides an easy way to eliminate these problems. It can be setup by almost anyone anywhere and usually requires only minor modifications to your configuration files.

Free!

Stumps is free! We created this tool as a way to end the frustration of our own team, and wanted to pass it along for everyone to use. The source is under the Apache 2.0 license which allows you to do whatever you want with it. Just remember, if you do something great, please send us your patches so we can keep enhancing Stumps for everyone.

Dashboard

Stumps can serve up multiple websites at the same time, and they can all be managed from a single web-based dashboard.

Adding Websites is Easy

To start serving up a new website, simply type in the domain name and click on the "Create" button. It's that easy.

Record Traffic

Once you have created a new website, click on the "record" button and Stumps will start capturing any traffic sent to that website.

Create Your Stump

After you've captured traffic, choose any of the requests, and use it to create a new stump.
